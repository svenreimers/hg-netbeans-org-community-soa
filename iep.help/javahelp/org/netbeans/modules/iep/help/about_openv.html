IEP operators are used in conjunction with the following data formats:

  • Table. A finite collection of events that belong to the same schema.
  • Stream. A collection of events that belong to the same schema, and that have timestamps.
  • Relation. A collection of tables that have the same schema, and that are all indexed by time.

Windows, also known as sliding windows, enable you to conceive of a meaningful boundary on the event stream so that you can query it effectively.

Operators and Schemas

You typically provide a schema definition in the property editor for the Stream Input operator, by providing an output schema name and attribute name, data type and size information.

Many operators that follow the Stream Input operator do not alter your schema definition, as these operators do not modify data or its format as it moves through the Intelligent Event Processing Module.

Exceptions to this rule include the following modifier operators:

  • Relation Map
  • Relation Aggregator
  • Tuple Based Aggregator
  • Time Based Aggregator
  • Tuple Serial Correlation
  • Stream Project and Filter

Operators that follow any of these modifiers inherit their new output schema.
